Comment 편집 UI — 방식 비교

테이블/컬럼 comment를 어떻게 편집할지 선택해주세요

A 인라인 항상 표시
테이블/컬럼 바로 아래에 comment 입력란이 항상 노출
PK id uuid ×
email varchar ×
created_at timestamp ×
특징: 항상 보이므로 comment 존재 여부 한눈에 파악.
테이블 노드가 세로로 길어짐. comment 없는 컬럼도 빈 줄 차지.
B 💬 아이콘 → 인라인 펼치기
아이콘 클릭 시 해당 항목 아래에 입력란 펼침. comment 없으면 노드 크기 유지
테이블 comment
PK id uuid ×
컬럼 comment
email varchar ×
created_at timestamp ×
특징: comment 없으면 노드가 컴팩트하게 유지.
💬 아이콘 색상으로 comment 존재 여부 구분. 클릭 한 번 더 필요.
C 우측 속성 패널
컬럼 클릭 시 우측에 속성 패널 오픈. comment 포함 모든 속성 한곳에서 편집
PK id uuid
email varchar
created_at timestamp
컬럼 속성 — id
이름
id
타입
uuid
Comment
제약조건
PK · NOT NULL
특징: 노드 자체는 깔끔, 모든 속성을 한 패널에서 편집.
화면 공간 필요. nullable/default 같은 컬럼 속성도 패널로 이동 가능.
