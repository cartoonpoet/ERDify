INDEX 지정 방식 선택

확정 레이아웃: PK · FK · NULL · UNIQUE · 논리명 · 이름 · 타입 · 삭제 — INDEX는?

A
컬럼 행에 IDX 체크박스
PK · FK · NULL · UQ · IDX · 논리명 · 이름 · 타입 · 삭제
체크하면 해당 컬럼에 단일 인덱스 생성
orders 주문 테이블 × 삭제
PK FK NULL UQ IDX 논리명 컬럼명 타입
특징: 단순. 체크 하나로 단일 컬럼 인덱스 바로 지정.
복합 인덱스(여러 컬럼) 지정 불가. DDL로 CREATE INDEX 자동 생성.
B
테이블 하단 인덱스 섹션
PK · FK · NULL · UQ · 논리명 · 이름 · 타입 · 삭제
테이블 아래 별도 섹션에서 인덱스 이름·컬럼 지정. 복합 인덱스 가능
orders 주문 테이블 × 삭제
PK FK NULL UQ 논리명 컬럼명 타입
INDEXES
user_id INDEX ×
status, created_at INDEX ×
reference_no UNIQUE ×
특징: 복합 인덱스, 이름 지정, UNIQUE 인덱스 등 세밀한 제어 가능.
설정이 약간 더 많음. 컬럼 행은 더 깔끔.
