C. 라우트 중심 코로케이션

섹션 컴포넌트를 해당 페이지 폴더 안에 바로 co-locate. 페이지 전용 섹션은 외부에서 import 불가. depth는 4지만 폴더 의미가 명확.

💡 추천: B — pages/components 분리가 가장 직관적이고, depth가 4 이상 깊어지지 않아. CSS 파일은 각 컴포넌트 파일 바로 옆에 두면(OrgRail.tsx + org-rail.css.ts) 폴더 없이도 코로케이션 가능.
